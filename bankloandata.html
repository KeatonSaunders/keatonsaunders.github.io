Classification of bank loan data using KNN and Logistic Regression

This project makes use of the Sci-kit learn machine learning library in Python. Preprocessing is used to encode categorical variables and standardize all variables. The KNeighborsClassifier and LogisticRegression models are then used to classify the data into 'good' and 'bad' loan applications. Because the data is labeled, we are able to use the Sci-kit learn classification report to compare our model's predictions against the labels in the original dataset. The project uses German bank loan data from the UCI Machine Learning Library, a description of which can be viewed here: UCI German Dataset

This project begins by importing the data and doing some basic exploratory data analysis using Seaborn to get a feel for what we are working with (above). The categorical variables are then encoded using sklearn's preprocessing module, as the model cannot handle the categorical variables in their raw form. The variables are then standardized using sklearn's StandardScaler before using train_test_split to split the data into training and testing subsets respectively. A testing size of 30% of the original dataset is chosen, as is common practice. The KNeighborsClassifier model is then fit to the training data, starting with a k-value of 1 neighbour. Following this, the optimal k-value needs is found by calculating the error rate (the percentage of predicted y-values which are not equal to the actual testing y-values) for each value of k between 1 and 40. This is shown below:

Based on the above graph, an optimal k-value of 12 is chosen. The model is then retrained using the new k-value and the classification report yields the following results:

Precision Recall f1-score Support
0 0.68 0.43 0.53 90
1 0.79 0.91 0.85 210
accuracy - - 0.77 300
macro-avg 0.74 0.67 0.69 300
weighted-avg 0.76 0.77 0.75 300

In order to understand the above results it is useful to know the following:

  • Precision = TP/(TP + FP)
  • Recall = TP/(TP+FN)
  • F1 Score = 2*(Recall * Precision) / (Recall + Precision)
  • Accuracy = Number of correct predicitions / Total predictions

    • Where:

  • True positive: actual = 1, predicted = 1
  • False positive: actual = 0, predicted = 1
  • False negative: actual = 1, predicted = 0
  • True negative: actual = 0, predicted = 0
